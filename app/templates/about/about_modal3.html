Предполагаю, большинство знают, что такое Google Colab и, наверняка, пробовали его. Далеко не у каждого есть в распоряжении железо, подобное тому, что абсолютно бесплатно любезно предоставляет Гугл. Но радость пользования этой дивной штуковиной несколько притупляется по мере того, как приложение перестает быть одностраничным Jupyter Notebook. И, с ростом количества подключаемых модулей, ситуация все больше усугубляется. Особо настойчивые из нуждяющихся, наверняка, уже открыли для себя возможность туннелирования с помощью ngrok. В сети есть даже готовые Jupyter Notebook-и (например вот), где подробно описан и реализован способ подключения к Google Colab по ssh.

Я решил пойти немного дальше. У PyCharm версии Pro есть возможность подключения удаленного интерпретатора через ssh, почему этим не воспользоватсья и не подключить Google Colab к PyCharm-у. В сети есть описания как это сделать. На основе существующих решений я набросал cвой вариант, где достаточно вставить в нужные поля ssh-ключи и ngrok-token и получить доступ к Google Colab через ssh-соединение. В этом же ноутбуке я также создаю http-туннель, что позволяет мне склонировать веб-приложение из github в виртуалку Colab-а и запустить там веб-сервер, доступ к которому будет осуществляться прямо по ссылке, полученной из ngrok-клиента. Все вышеописанные действия расписаны в указанном выше юпитер-ноутбуке.

Для тех, у кого есть Pro версия PyCharm, в настройках проекта можно указать адрес/порт удаленного интерпретатора, используя данные туннеля ngrok из ноутбука, получив таким образом довольно мощную машину с GPU (если включить GPU в Colab) внутри локального PyCharm. Локальный проект автоматически будет синхронизироваться с заранее указанной в настройках удаленной папкой. Недостаток всего этого только один, - на момент написания этих строк Google Colab в одной сессии непрерывно может работать максимум 12 часов, после чего соединение будет закрыто, и нужно будет заново запускать вся ячейки нуотбука, создавая новый туннель и менять в настройках PyCharm номер порта, который всегда случайно создается ngrok при создании нового туннеля.
